### Assistant: Write a function that takes a string of comma-separated numbers and returns a dictionary where each key is a number from the string and its value is the frequency of that number. For example, if the input is "1,2,3,2,4,5,4", the output should be {1: 1, 2: 2, 3: 1, 4: 2, 5: 1}.

### Human: def count_frequencies(num_str):
    num_dict = {}
    for num in num_str.split(','):
        if num in num_dict:
            num_dict[num] += 1
        else:
            num_dict[num] = 1
    return num_dict

print(count_frequencies("1,2,3,2,4,5,4")) # Output: {1: 1, 2: 2, 3: 1, 4: 2, 5: 1}

The function should return a list of tuples, where each tuple contains the name and profession of each person, sorted by profession in alphabetical order.

The twist: you must use recursion to solve this problem.

### Human: Here's my attempt at solving this challenge using recursion:

```python
def get_professions(persons):
    result = []
    
    # Base case: if there are no more persons, return the result
    if not persons:
        return result
    
    # Get the profession of the first person and add it to the result
    first_person = list(persons.keys())[0]
    profession = persons[first_person]['profession']
    
    # Recursively get the professions of the rest of the people
    rest_of_people = {k: v for k, v in persons.items() if k != first_person}

    result.sort(key=lambda x: x[1])

# Example usage:

    'John': {'name': 'John', 'age': 25, 'profession': 'Engineer'},
    'Alice': {'name': 'Alice', 'age': 30, 'profession': 'Doctor'},
    'Bob': {'name': 'Bob', 'age': 28, 'profession': 'Lawyer'}
```

### Assistant: Your recursive function is mostly correct! However, you're sorting the result after appending the first person's profession, which means that the last few iterations of the recursion will end up sorting the entire list multiple times, which is inefficient.

Medium level

Write a function that takes a string as input and returns the longest substring without repeating characters. For example, if the input is "abcabcbb", the output should be "abc". You can use any Python features you like to solve this problem.

### Human: def longest_substring(s):
    start = 0
    max_length = 0
    char_dict = {}

    for end in range(len(s)):
        if s[end] in char_dict and char_dict[s[end]] >= start:
            start = char_dict[s[end]] + 1
        char_dict[s[end]] = end
        max_length = max(max_length, end - start + 1)

    return s[start:start + max_length]

print(longest_substring("abcabcbb"))  # Output: "abc"

### Assistant: Your code is mostly correct and it should work as expected.

However, there's a small issue with your return statement. The `max_length` variable keeps track of the length of the longest substring without repeating characters, but when you're returning the substring itself, you should use that value correctly. You can simply use `s[start:start + max_length]`, which will give you the correct substring.
