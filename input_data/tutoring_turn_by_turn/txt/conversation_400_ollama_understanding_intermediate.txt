Here's a HARD question:

Write a function called `merge_sorted_lists` that takes two sorted lists as input and returns a new sorted list containing all elements from both inputs. The function should have a time complexity of O(n + m), where n and m are the lengths of the input lists.

### Human: Here's a Python solution for the problem:

```python
def merge_sorted_lists(list1, list2):
    result = []
    while len(list1) > 0 and len(list2) > 0:
        if list1[0] <= list2[0]:
            result.append(list1.pop(0))
        else:
            result.append(list2.pop(0))
    return result + list1 + list2
```

This function merges two sorted lists into a new sorted list. It does this by repeatedly taking the smallest element from either of the input lists and adding it to the result list, until one of the input lists is empty. At that point, all remaining elements in the non-empty list are appended to the result. The time complexity of this function is O(n + m), because each element is visited once (either by being popped off a list or being added at the end).
