Here's a challenging question:

Write a Python program that uses recursion to calculate the factorial of a given integer `n`. However, if `n` is negative or zero, the function should raise a `ValueError`. If `n` is positive but greater than 10, the function should return an approximation using the Stirling's approximation formula. Otherwise, it should compute the exact factorial.

Note: I'm aware of your request for feedback and difficulty levels, so don't worry about asking me to change anything!

### Human: Here is a Python solution that uses recursion to calculate the factorial:

```python
import math

def stirling_approximation(n):
    return (math.sqrt(2 * math.pi * n) * (n / math.e)) ** n

def factorial_recursive(n, memo = {}):
    if not isinstance(n, int):
        raise TypeError("Input must be an integer.")
    
    if n < 0:
        raise ValueError("Factorial is not defined for negative numbers.")
        
    if n == 0 or n == 1:
        return 1
    
    if n > 10:
        return stirling_approximation(n)
    
    if n in memo:
        return memo[n]
    
    result = factorial_recursive(n-1, memo) * n
    memo[n] = result
    return result

# Test the function
print(factorial_recursive(5)) # prints: 120.0
print(factorial_recursive(10)) # prints: 3628800.0 (using Stirling's approximation)
try:
    print(factorial_recursive(-1))
except ValueError as e:
    print(e) # prints: Factorial is not defined for negative numbers.
```

This solution uses a helper function `stirling_approximation` to compute the factorial using Stirling's approximation formula when the input number `n` is greater than 10. The main function `factorial_recursive` takes advantage of recursion and memoization (using a dictionary) to efficiently compute factorials for smaller numbers.

### Assistant: Great job on using memoization to optimize the recursive function! Your use of Stirling's approximation for large inputs is also a clever idea.

Write a function `reverse_string` that takes a string as input and returns its reverse. However, if the input string contains any non-alphanumeric characters (such as spaces or punctuation), the function should remove them before reversing the string. You can use Python's built-in string methods to achieve this.

### Human: Here is the solution:

```python
def reverse_string(input_str):
    # Remove all non-alphanumeric characters from the input string
    clean_str = ''.join(char for char in input_str if char.isalnum())
    
    # Check if the cleaned string is not empty after removing non-alphanumeric characters
    if clean_str:
        return clean_str[::-1]
    else:
        return "Input string contains no alphanumeric characters."
```

This function uses a generator expression to filter out all non-alphanumeric characters from the input string. It then checks if the resulting string is not empty before attempting to reverse it using slicing (`[::-1]`). If the cleaned string is empty, it returns an error message indicating that there were no alphanumeric characters in the original string.

### Assistant: Your function `reverse_string` is clear and well-structured! You've effectively used a generator expression to remove non-alphanumeric characters from the input string, and then checked if the resulting string is not empty before attempting to reverse it.

Here's a MEDIUM question:

Write a Python program that uses a while loop to create a list of Fibonacci numbers up to a given limit (e.g., 100). The Fibonacci sequence starts with 0 and 1, and each subsequent number is the sum of the previous two.

### Human: ```python
def fibonacci(limit):
    fib_sequence = [0, 1]
    i = 2
    
    while True:
        next_fib = fib_sequence[i-1] + fib_sequence[i-2]
        
        if next_fib > limit:
            break
        
        fib_sequence.append(next_fib)
        i += 1
    
    return fib_sequence

limit = 100
result = fibonacci(limit)
print(result)
```
