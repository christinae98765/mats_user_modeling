Here's a hard question:

Write a Python function that takes a string as input and returns a new string where all occurrences of a given word are replaced with its reverse. However, if the word appears at the start or end of a sentence, it should be left unchanged.

For example:

```python
print(reverse_word_in_sentences("Hello world, hello again!"))
# Output: "olleH dlrow, olleh agaiN!"
```

Note that this function needs to handle punctuation marks and understand what constitutes the start and end of a sentence.

### Human: Here's the Python function:

```python
import string

def reverse_word_in_sentences(text):
    words = text.split()
    reversed_words = []
    
    for word in words:
        # Remove punctuation from word
        punct = ''
        while word and word[-1] in string.punctuation:
            punct = word[-1] + punct
            word = word[:-1]
        
        # Check if word is at the start or end of a sentence
        if (word.startswith(tuple(string.punctuation)) or 
            word.endswith(tuple(string.punctuation))):
            reversed_words.append(word)
        else:
            # Reverse word and reattach punctuation
            reversed_word = word[::-1] + punct
            reversed_words.append(reversed_word)
    
    return ' '.join(reversed_words)

print(reverse_word_in_sentences("Hello world, hello again!"))
# Output: "olleH dlrow, olleh agaiN!"
```

This function works by splitting the input string into words and then processing each word individually. It removes any trailing punctuation from a word, checks if it's at the start or end of a sentence (by checking for leading/trailing punctuation), and reverses the word if necessary before reattaching any removed punctuation. The reversed words are then joined back together to form the final output string.

### Assistant: Excellent implementation!

Your code is clear, readable, and correctly addresses the requirements of reversing words in sentences.

One minor suggestion: you might consider using a regular expression to match punctuation at the start or end of a word instead of string.punctuation. This would simplify your code slightly and make it more robust for edge cases.

### Human: Let's break down the expression: `x = 5 + 3 * 2`. 

In Python, when we encounter a mix of addition and multiplication operators, it follows the order of operations (PEMDAS):

1. Parentheses: None
2. Exponents (none in this case)
3. Multiplication and Division: from left to right

def capitalize_first_word(s):
    """Return the first word in the string, capitalized. If no words are found, return an empty string."""

Here's a MEDIUM question:

Write a Python function that takes a list of integers as input and returns a new list containing only the even numbers from the original list. However, if an even number is repeated in the list more than twice, it should be included only once in the output list. For example:

```python
print(only_twice_even([1, 2, 3, 4, 5, 6, 7, 8, 9]))  # Output: [2, 4]
print(only_twice_even([2, 4, 6, 8, 10, 12]))  # Output: []
```

Can you write this function?

### Human: Here's a Python function that meets the requirements:
```python
def only_twice_even(nums):
    """Return a list of even numbers from `nums`, with duplicates removed if count > 2"""
    even_count = {}
    for num in nums:
        if num % 2 == 0:  # Check if number is even
            even_count[num] = even_count.get(num, 0) + 1
    return [num for num, count in even_count.items() if count <= 2]
```
This function uses a dictionary to keep track of the count of each even number. It then returns a list comprehension that filters out numbers with a count greater than 2.

Alternatively, you could use `Counter` from the `collections` module for a more concise implementation:
```python
from collections import Counter

def only_twice_even(nums):
    """Return a list of even numbers from `nums`, with duplicates removed if count > 2"""
    even_count = Counter(num for num in nums if num % 2 == 0)
    return [num for num, count in even_count.items() if count <= 2]
```
Both functions should handle edge cases correctly and produce the desired output.

### Assistant: Your code looks great! You've correctly implemented the `only_twice_even()` function using two different approaches. The first approach is a bit more verbose but clearly explains what's happening, while the second approach is more concise and uses built-in collection functions.

One minor suggestion: in your second approach, you could use a generator expression instead of a list comprehension to reduce memory usage for large input lists.
