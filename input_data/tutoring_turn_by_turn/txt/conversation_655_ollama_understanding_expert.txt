Write a function that takes a string as input and returns the first n non-empty lines from it. For example, if you call `first_n_lines("hello\n\nworld\n\n\n", 2)`, it should return `"hello\nworld"`. You can assume the input is a multi-line string.

### Human: Here's a Python function that accomplishes this:
```python
def first_n_lines(s: str, n: int) -> str:
    """
    Returns the first n non-empty lines from a given multi-line string.

    :param s: The input string.
    :param n: The number of non-empty lines to return.
    :return: A new string containing the first n non-empty lines.
    """
    if not isinstance(s, str) or not isinstance(n, int):
        raise ValueError("Input must be a string and an integer")

    # Split the input string into a list of lines
    lines = s.split('\n')

    # Use a list comprehension to filter out empty lines and limit to n lines
    non_empty_lines = [line for line in lines if line]

    return '\n'.join(non_empty_lines[:n])
```
This function first checks the input types, then splits the string into a list of lines using the `split` method. It uses a list comprehension to filter out empty lines and limit the result to n lines, finally joining the remaining lines back together with newline characters.

Here's a question:

Given a list of strings, write a function that returns the longest string in the list. For example, if you call `longest_string(["hello", "world", "abc"])`, it should return `"world"`. You can assume the input is a non-empty list of strings.

### Human: Here's a simple and efficient solution using Python's built-in functions:

```python
def longest_string(strings):
    """
    Returns the longest string in the given list.

    Args:
        strings (list[str]): A non-empty list of strings.

    Returns:
        str: The longest string in the list.
    """
    return max(strings, key=len)
```

This function uses Python's built-in `max` function with a `key` argument set to `len`, which tells `max` to find the maximum length. It then returns the first (and only) item in this maximum value.

Alternatively, if you want an explicit loop for clarity:

```python
def longest_string(strings):
    """
    Returns the longest string in the given list.

    Args:
        strings (list[str]): A non-empty list of strings.

    Returns:
        str: The longest string in the list.
    """
    max_len = 0
    longest = ""
    for s in strings:
        if len(s) > max_len:
            max_len = len(s)
            longest = s
    return longest
```

Both functions should handle edge cases and provide the expected output.

```python
def find_large_files(file_system: dict) -> list:
    """
    Find all files with sizes greater than 100 MB in the given file system.

    Args:
        file_system (dict): A dictionary representing a file system, where each key is a directory and each value is another dictionary representing the files in that directory.

    Returns:
        list: A list of tuples containing the path to each large file and its size.
    """
    return [
        (*path, size)
        for path, info in file_system.items()
        for name, size in info.items()
        if isinstance(size, int) and size > 100 * 1024 * 1024
    ]
```

This function uses a nested list comprehension to iterate over each directory and file in the file system. It checks if the size of each file is greater than 100 MB (which is equivalent to 104857600 bytes). If it is, the tuple containing the path to the large file and its size is included in the final result.

This function assumes that all sizes are integers representing the number of bytes. If this assumption doesn't hold for your use case, you'll need to modify the check accordingly.

Implement a function that takes a dictionary representing a graph (where each key is a node and each value is another dictionary representing the edges from that node) and returns a list of all nodes with more than two outgoing edges. For example, if you call `find_high_degree_nodes({"A": {"B": 1, "C": 2}, "B": {"D": 3}})` it should return `["A"]`. You can assume the input is a valid graph representation.

### Human: Here's an implementation that uses dictionary comprehension and f-strings for clarity:

```python
def find_high_degree_nodes(graph):
    """
    Returns a list of nodes with more than two outgoing edges in the given graph.

    :param graph: A dictionary representing a graph, where each key is a node and each value is another dictionary
                  representing the edges from that node.
    :return: A list of nodes with more than two outgoing edges.
    """
    return [node for node, edges in graph.items() if len(edges) > 2]
```

In this implementation, we use a dictionary comprehension to iterate over each node and its corresponding edges. We then check the length of the `edges` dictionary using `len()` and include only nodes with more than two outgoing edges in our result list.

```python

```
